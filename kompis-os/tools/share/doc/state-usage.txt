The `state` command used to automate state transitions for an entity
capture* <entity>                        Worktree → Snapshot Takes an atomic filesystem snapshot.
revert*  <entity> <fs-snapshot>          Snapshot → Worktree Rolls back the live state to a specific FS snapshot.
commit   <entity> <fs-snapshot>          Snapshot → Archive Promotes a FS snapshot into the CDC (Restic/Borg) repository.
prepare  <entity> <cdc-snapshot>         Archive → Snapshot Hydrates a CDC snapshot into a mountable/usable FS snapshot.
send     <entity> <cdc-snapshot> <host>  Archive → Replica Pushes the specific snapshot data to an offsite location.
retrieve <entity> <cdc-snapshot> <host>  Replica → Archive Fetches missing blobs/metadata reconstructs a snapshot in the local Archive.
clean*   <entity> <fs-snapshot>          Forces worktree to correspond exactly to a given snapshot
verify   <entity> <snapshot>             Physical Integrity
check    <entity> <snapshot>             Logical Compliance
prune    <entity> <layer>                Prune to align with policy
align    <entity> <layer>                Prune and backup to align policy

*invokes entity-specific hook
